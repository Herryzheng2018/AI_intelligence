## 1. 界面总览

AI情报官采用经典的「侧边栏 + 顶栏 + 主内容区」三段式布局，深色科技风格设计。

### 侧边导航栏

位于界面左侧，包含六个功能模块入口：

| 图标 | 模块 | 说明 |
|------|------|------|
| LayoutDashboard | 指挥中心 | 实时仪表盘，全局概览 |
| Satellite | 信息源管理 | 配置和管理信息采集源 |
| Newspaper | 情报流 | 实时查看采集到的情报 |
| FileText | AI 周报 | 自动生成情报周报 |
| Settings | 引擎设置 | 采集引擎与模型配置 |
| Network | 系统架构 | 查看四层架构详情 |

侧边栏底部显示「7x24 巡航中」状态卡片，实时展示当前监控的信息源数量和今日采集量。

### 顶部状态栏

位于界面顶部，包含：

- **应用标题**：AI情报官 · 7x24 智能信息处理 Agent
- **实时时钟**：显示当前时间（HH:MM:SS 格式）
- **状态徽章**：显示系统运行状态（运行中 / 维护中）
- **通知图标**：点击查看系统通知
- **用户头像**：点击进入个人设置

### 主内容区

根据侧边栏选中的模块，显示对应的功能页面。每个页面顶部都有页面标题和简短描述。

## 2. 指挥中心（Dashboard）

指挥中心是系统的全局概览页面，一屏掌握所有关键信息。

### 2.1 统计卡片

页面顶部显示四个核心统计卡片：

| 卡片 | 含义 | 数据来源 |
|------|------|----------|
| 今日采集 | 今日已采集的情报总数 | 实时统计 |
| 高价值情报 | 经 LLM 评分 ≥ 70 的情报数 | 语义筛选 |
| 活跃信息源 | 当前正在监控的信息源数 | 信息源管理 |
| 引擎负载 | 四种采集引擎的平均负载 | 引擎调度 |

每个卡片右侧显示与昨日相比的变化趋势（↑上升 / ↓下降）。

### 2.2 采集趋势图

以面积图展示最近 24 小时的情报采集趋势，横轴为时间，纵轴为采集量。绿色面积代表高价值情报，蓝色面积代表普通情报。鼠标悬停可查看具体时间点的数据。

### 2.3 信息源分布

饼图展示各类型信息源（RSS、微信公众号、掘金、知乎、谷歌学术、GitHub、Web）的情报贡献占比。点击图例可隐藏/显示对应类型。

### 2.4 引擎调度分布

柱状图展示四种采集引擎（HTTPX、Patchright、CDP、Crawl4AI）的调度次数，帮助了解引擎使用情况。

### 2.5 实时活动日志

页面右侧的滚动列表，实时显示系统活动，包括：

- **时间戳**：精确到秒
- **日志级别**：success（绿色）/ info（蓝色）/ warning（黄色）/ error（红色）
- **引擎**：使用的采集引擎
- **信息源**：来源名称
- **消息**：活动描述

日志每 3-5 秒自动更新，最新日志显示在顶部。

### 2.6 四层架构状态

展示「感知·认知·记忆·表达」四层架构的实时运行状态，包括：

- 层级名称与图标
- 运行状态（运行中 / 空闲 / 异常）
- 负载百分比与进度条

### 2.7 Top 信息源排行

显示采集量最高的 5 个信息源，包括名称、类型图标、采集量、高价值数量。

## 3. 信息源管理（Sources）

信息源管理模块用于配置和管理所有信息采集源。

### 3.1 信息源列表

以表格形式展示所有信息源，每行包含：

| 列名 | 说明 |
|------|------|
| 状态 | 彩色圆点（绿色=活跃 / 黄色=空闲 / 红色=异常 / 灰色=已暂停） |
| 名称 | 信息源名称 |
| 类型 | 类型图标与标签（RSS/微信/掘金等） |
| URL | 信息源地址 |
| 引擎 | 分配的采集引擎 |
| 间隔 | 采集间隔（分钟） |
| 已采集 | 累计采集情报数 |
| 高价值 | 高价值情报数 |
| 最后采集 | 上次采集时间 |
| 操作 | 编辑 / 暂停 / 删除按钮 |

### 3.2 筛选与搜索

页面顶部提供筛选工具栏：

- **搜索框**：按名称或 URL 搜索信息源
- **类型筛选**：下拉选择特定类型（RSS/微信/掘金等）
- **状态筛选**：下拉选择特定状态（活跃/空闲/异常/已暂停）

### 3.3 添加信息源

点击右上角「添加信息源」按钮，弹出对话框：

1. **名称**：为信息源起一个易识别的名字
2. **类型**：选择信息源类型
   - RSS：博客、新闻源的 RSS 订阅地址
   - 微信公众号：公众号文章列表
   - 掘金：掘金网站专栏
   - 知乎：知乎话题或专栏
   - 谷歌学术：学术论文搜索
   - GitHub：仓库 Release 或 Issue
   - Web：通用网页
3. **URL**：填写信息源地址（如 `https://hnrss.org/frontpage`）
4. **采集引擎**：选择采集引擎（详见第 6 节）
5. **采集间隔**：设置采集频率（分钟），建议 30-120 分钟

填写完成后点击「保存」，新信息源立即开始采集。

### 3.4 编辑信息源

点击信息源行的「编辑」图标，可修改名称、URL、引擎、间隔等配置。

### 3.5 暂停/恢复采集

点击「暂停」图标可暂停该信息源的采集任务，状态变为灰色。再次点击「恢复」可重新启动采集。

### 3.6 删除信息源

点击「删除」图标可移除信息源。删除前会弹出确认对话框，防止误操作。

> **提示**：删除信息源不会删除已采集的情报数据。

## 4. 情报流（Intelligence Stream）

情报流模块实时展示所有采集到的情报，支持多维筛选和详情查看。

### 4.1 情报列表

以卡片流形式展示情报，每张卡片包含：

- **标题**：情报标题（点击可查看详情）
- **AI 摘要**：LLM 生成的 100 字核心摘要
- **来源徽章**：信息源名称与类型图标
- **引擎徽章**：使用的采集引擎
- **时间徽章**：采集时间
- **语义评分**：右上角彩色徽章，显示 0-100 分
  - 绿色（≥80）：极高价值
  - 青色（70-79）：高价值
  - 黄色（50-69）：中等价值
  - 灰色（<50）：普通信息
- **标签**：情报的主题标签（如 #AI、#论文、#开源）

### 4.2 筛选工具栏

页面顶部提供强大的筛选功能：

- **搜索框**：按标题或摘要关键词搜索
- **来源筛选**：按信息源类型筛选
- **评分筛选**：按语义评分筛选（全部 / 高价值 / 极高价值）
- **引擎筛选**：按采集引擎筛选
- **自动刷新开关**：开启后每 10 秒自动刷新情报列表

### 4.3 查看情报详情

点击情报卡片，右侧滑出详情面板，展示：

- 完整标题与摘要
- 所有元数据（来源、引擎、时间、评分）
- 主题标签
- 「查看原文」按钮（在新标签页打开原始链接）

### 4.4 情报评分说明

每条情报都经过 LLM 语义评分，评分标准：

| 评分区间 | 等级 | 含义 |
|----------|------|------|
| 80-100 | 极高价值 | 与关注领域高度相关，含重要信息 |
| 70-79 | 高价值 | 与关注领域相关，值得阅读 |
| 50-69 | 中等价值 | 部分相关，可选择性阅读 |
| 0-49 | 普通 | 相关性较低，仅作存档 |

> 默认情况下，评分 ≥ 70 的情报会被标记为「高价值」并优先展示。

## 5. AI 周报（Weekly Reports）

AI 周报模块自动将一周内的高价值情报整合成结构化周报。

### 5.1 周报列表

左侧列表展示所有已生成的周报，每条包含：

- **周报标题**：如「AI 情报周报 · 第 27 周」
- **统计周期**：如 2026-06-28 至 2026-07-04
- **生成时间**：周报生成的时间戳
- **统计信息**：事件组数、情报总数、高价值情报数
- **状态徽章**：已完成 / 生成中 / 已排期

点击列表项可在右侧查看周报内容。

### 5.2 周报内容预览

右侧区域展示选中周报的完整内容，采用 Markdown 渲染：

- **一级标题**：周报标题
- **二级标题**：各事件分组（如「大模型动态」「开源项目」「学术论文」）
- **三级标题**：具体事件
- **正文**：深度分析与综述
- **列表**：相关情报条目

### 5.3 生成新周报

点击右上角「生成新周报」按钮，触发 map-reduce 流水线：

#### 流水线可视化

生成过程中，页面顶部显示四阶段进度条：

| 阶段 | 名称 | 说明 | 耗时 |
|------|------|------|------|
| 1 | Map 原子化 | 调用 LLM 为每条情报生成 100 字核心摘要 | ~1.5s |
| 2 | Shuffle 聚类 | 语义聚类，合并重复报道 | ~1.5s |
| 3 | Reduce 综述 | 生成深度分析段落 | ~1.5s |
| 4 | 拼接 Markdown | 组装最终周报 | ~1s |

每个阶段完成后，对应的进度条变为绿色并显示对勾。全部完成后，新周报自动添加到列表顶部。

#### 生成原理

```
原始情报（100+ 条）
    ↓ Map 阶段
原子摘要（每条 100 字）
    ↓ Shuffle 阶段
语义聚类（合并重复）
    ↓ Reduce 阶段
深度综述（按主题分组）
    ↓ 拼接阶段
最终 Markdown 周报
```

### 5.4 周报统计

每份周报生成后，系统会统计：

- **事件组数**：聚类后的事件主题数量
- **情报总数**：纳入周报的情报条数
- **高价值情报数**：评分 ≥ 70 的情报数

### 5.5 自动生成设置

在「引擎设置 → 模型配置」中可开启「每周自动生成周报」，系统会在每周日 23:00 自动触发周报生成。

## 6. 引擎设置（Engines）

引擎设置模块用于配置四种采集引擎和 LLM 模型。

### 6.1 采集引擎配置

系统提供四种采集引擎，各有适用场景：

#### HTTPX 请求引擎

- **适用场景**：传统 CMS、静态博客、RSS 订阅
- **特点**：速度最快，资源消耗最低
- **配置项**：超时时间、重试次数、并发数
- **建议**：优先用于 RSS 和静态网站

#### Patchright 隐匿渲染引擎

- **适用场景**：动态网页、新闻门户、需要 JavaScript 渲染的站点
- **特点**：基于 CDP 指纹修补，绕过 Cloudflare 等反爬检测
- **配置项**：渲染等待时间、隐匿等级、User-Agent
- **建议**：用于动态加载内容的网站

#### CDP 环境接管引擎

- **适用场景**：长身份验证社交媒体（微信公众号、知乎登录态页面）
- **特点**：复用本地浏览器登录态，无需重新登录
- **配置项**：浏览器路径、用户数据目录、接管端口
- **建议**：用于需要登录才能访问的站点

#### Crawl4AI 智能解析引擎

- **适用场景**：文档、论文、长文章
- **特点**：DOM 转 Markdown，智能分块，保留结构
- **配置项**：分块大小、解析模式、输出格式
- **建议**：用于学术论文和技术文档

### 6.2 引擎开关

每个引擎卡片右侧有开关按钮，可快速启用/禁用引擎。禁用后，分配给该引擎的信息源会自动切换到备用引擎。

### 6.3 引擎统计

每个引擎卡片显示运行统计：

- **总请求数**：累计发起的采集请求数
- **成功率**：成功采集的比例
- **平均延迟**：单次采集的平均耗时

### 6.4 模型配置

「模型配置」标签页用于设置 LLM 模型：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 模型提供商 | LLM 服务商 | SiliconFlow / OpenAI / DeepSeek / Anthropic |
| 模型名称 | 具体模型 | Qwen2.5-72B-Instruct / GPT-4o / Claude-3.5-Sonnet |
| API Key | 模型访问密钥 | sk-xxxxxxxxxxxx |
| 评分阈值 | 高价值情报的评分门槛 | 70（默认） |

#### 推荐模型配置

| 用途 | 推荐模型 | 说明 |
|------|----------|------|
| 语义筛选 | Qwen2.5-72B-Instruct | 性价比高，中文理解强 |
| 周报生成 | DeepSeek-V3 | 长文本综述能力强 |
| 高质量分析 | GPT-4o / Claude-3.5-Sonnet | 综合能力最强 |

### 6.5 存储设置

「存储设置」标签页用于配置数据存储：

- **存储模式**：
  - SQLite WAL（默认）：单文件数据库，适合中小规模
  - PocketBase：嵌入式数据库，支持 REST API
- **自动周报**：开启后每周日 23:00 自动生成周报
- **数据保留**：设置情报数据的保留天数（默认 90 天）

## 7. 系统架构（Architecture）

系统架构模块展示 AI情报官的四层分布式架构设计。

### 7.1 四层架构总览

系统采用「感知·认知·记忆·表达」四层架构：

#### L1 感知层（Perception）

负责信息采集，包含：

- **HTTPX 请求模块**：传统 HTTP 请求，技术栈 HTTPX + BeautifulSoup
- **Patchright 隐匿渲染模块**：动态网页渲染，技术栈 Patchright + CDP
- **CDP 环境接管模块**：登录态复用，技术栈 Chrome DevTools Protocol
- **Crawl4AI 智能解析模块**：文档解析，技术栈 Crawl4AI + DOM-to-Markdown

#### L2 认知层（Cognition）

负责语义理解与筛选，包含：

- **LLM 语义评分模块**：对每条情报打分（0-100），技术栈 LLM API
- **关注点匹配模块**：与用户关注领域匹配，技术栈 Embedding + 相似度
- **去重聚类模块**：语义去重，技术栈 向量数据库

#### L3 记忆层（Memory）

负责数据存储，包含：

- **原始情报存储**：SQLite WAL 模式，支持并发读写
- **向量索引存储**：语义检索，技术栈 向量数据库
- **元数据管理**：来源、时间、评分等元信息

#### L4 表达层（Expression）

负责情报输出，包含：

- **Map-Reduce 周报生成**：原子化 → 聚类 → 综述 → 拼接
- **Markdown 渲染**：周报内容渲染
- **实时推送**：高价值情报即时通知

### 7.2 数据流

```
信息源 → 感知层（采集） → 认知层（评分筛选） → 记忆层（存储）
                                                        ↓
用户 ← 表达层（周报/推送） ← 记忆层（检索） ← 认知层（聚类）
```

### 7.3 核心优势

- **主动巡航**：7x24 小时不间断监控，无需人工触发
- **语义筛选**：LLM 理解内容含义，而非关键词匹配
- **多引擎调度**：根据信息源特性智能选择采集方式
- **自动闭环**：从采集到周报全流程自动化

### 7.4 参考项目

- Wiseflow：通用信息提取框架
- Crawl4AI：智能网页爬取
- Patchright：隐匿浏览器自动化

## 8. 移动端使用

AI情报官完整支持移动端访问，自适应布局。

### 8.1 访问方式

在手机浏览器输入应用地址（如 `http://your-server:3000`）即可访问。

### 8.2 移动端布局

- **顶部栏**：显示应用名称、汉堡菜单按钮、时钟
- **汉堡菜单**：点击左上角按钮展开侧边导航
- **主内容区**：单列布局，卡片自适应宽度

### 8.3 移动端操作

- 点击汉堡菜单切换导航模块
- 上下滑动浏览情报列表
- 点击情报卡片查看详情（全屏展开）
- 支持双指缩放查看图表细节

## 9. 快捷操作

### 9.1 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `1-6` | 快速切换六个功能模块 |
| `/` | 聚焦搜索框 |
| `Esc` | 关闭对话框/详情面板 |
| `R` | 刷新当前页面数据 |

### 9.2 操作建议

#### 新用户快速上手流程

1. 进入「信息源管理」，添加 3-5 个感兴趣的信息源
2. 进入「引擎设置」，配置 LLM 模型（API Key）
3. 等待 5-10 分钟，让系统完成首轮采集
4. 进入「指挥中心」查看采集概览
5. 进入「情报流」浏览高价值情报
6. 累积一周后，进入「AI 周报」生成首份周报

#### 日常使用建议

- 每日查看「指挥中心」了解采集概况
- 在「情报流」中按评分筛选高价值情报阅读
- 每周查看「AI 周报」获取领域动态综述
- 定期在「信息源管理」中调整信息源配置
- 根据引擎统计优化采集引擎分配

#### 性能优化建议

- RSS 信息源优先使用 HTTPX 引擎（速度最快）
- 动态网站使用 Patchright 引擎（兼容性好）
- 学术论文使用 Crawl4AI 引擎（解析精准）
- 需要登录的站点使用 CDP 引擎（复用登录态）
- 采集间隔建议设置为 30-120 分钟，避免过于频繁
